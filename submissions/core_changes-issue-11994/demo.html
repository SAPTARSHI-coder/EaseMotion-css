Issue #11994

Submission Validator Whitelist Bug

The validator currently allows top-level file changes despite the submission-only policy. This demo shows which files are wrongly permitted and how the fix resolves it.

Current Behavior (Broken)

The validator explicitly whitelists these 5 files — changes to them are not rejected, violating the policy:

FileCurrentlyShould Be
README.md⚠ Allowed (bug)Blocked
LICENSE⚠ Allowed (bug)Blocked
package.json⚠ Allowed (bug)Blocked
easemotion.css⚠ Allowed (bug)Blocked
easemotion.min.css⚠ Allowed (bug)Blocked

Root Cause (validator lines 127–133)

} else {
  if (
    filename !== 'README.md' &&
    filename !== 'LICENSE' &&
    filename !== 'package.json' &&
    filename !== 'easemotion.css' &&
    filename !== 'easemotion.min.css'
  ) {
    coreFiles.push(filename);  // ← only these are rejected
  }
  // ← the 5 whitelisted files fall through silently
}

These 5 files skip the coreFiles check entirely.

Proposed Fix

Remove the whitelist condition — reject ALL non-submission files:

} else {
  // ALL files outside submissions/ are core violations
  coreFiles.push(filename);
}
