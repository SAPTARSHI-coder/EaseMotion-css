Content-Visibility: Auto

Lazy rendering for off-screen sections — stable scrollbar, faster paint

Performance Note — This page has 15 sections with content-visibility: auto. Only visible cards are rendered. Scroll down to see new cards paint lazily.

1. What is content-visibility?

content-visibility: auto is a CSS property that tells the browser to skip rendering for elements outside the viewport. The element is still in the DOM and accessible, but layout, paint, and compositing are deferred until the user scrolls near it.

This can dramatically reduce initial page load time on content-heavy pages.

2. Why contain-intrinsic-size?

Without contain-intrinsic-size, elements with content-visibility: auto would have zero height before rendering, causing the scrollbar to grow as you scroll — a jarring experience.

contain-intrinsic-size: 300px reserves an estimated height so the scrollbar remains stable.

3. Render strategy

The browser uses proximity-based activation. When the element is within 1500px of the viewport (or a configured proximity amount), the browser renders it. This means content is ready by the time the user scrolls to it.

This is different from lazy loading for images — it affects the entire rendering pipeline.

4. Page structure planning

For maximum benefit, organize pages so that each logical section or card is a separate element with content-visibility: auto. Avoid putting it on the body or top-level wrapper — the viewport-visible area should render immediately.

Best applied to: list items, comment threads, long documentation, search results.

5. Interaction with contain

Adding contain: layout style paint alongside content-visibility: auto can further improve performance by isolating each card's subtree from the rest of the page. However, content-visibility: auto already implies contain: layout style paint size when the element is not visible.

6. Debugging visibility

In DevTools, elements with content-visibility: auto that have not yet rendered show a special badge. You can inspect the "Rendering" tab to see which elements are skipped. This is a great way to verify your optimization is working.

7. Common pitfalls

Do not use content-visibility: auto on elements that need to be measured immediately (e.g., for scroll-triggered animations). The element may report zero size before it is rendered.

Always pair with contain-intrinsic-size to avoid layout instability.

8. Impact on search

Content hidden with content-visibility: auto is still in the DOM and accessible to search engine crawlers. The property only affects rendering, not the accessibility tree or the DOM.

9. Long lists

For virtual-scrolling-like behavior on long lists, combine content-visibility: auto with contain-intrinsic-size set to the average item height. The browser will only render items near the viewport.

This works well for 100+ item lists without needing JavaScript virtualization.

10. Browser support

content-visibility is supported in Chromium-based browsers (Chrome, Edge, Opera, Brave, Samsung Internet). Firefox and Safari do not yet support it, but the property degrades gracefully — the page renders normally.

11. Comparison: with vs. without

Without content-visibility
  • All 15 sections render on initial load
  • Higher Largest Contentful Paint (LCP) time
  • More layout work for the browser
With content-visibility: auto
  • Only visible sections render initially
  • Significantly reduced LCP on long pages
  • Stable scrollbar via contain-intrinsic-size

12. contain-intrinsic-size values

The contain-intrinsic-size shorthand accepts one or two values. Use two values for width and height (e.g., contain-intrinsic-size: 100% 300px) or a single value for both.

For responsive layouts, the width can be set to 100% while only the height is estimated.

13. Progressive enhancement

Since content-visibility is not universally supported, treat it as a progressive enhancement. The page must work correctly without it. Use feature detection (e.g., @supports (content-visibility: auto)) if you need fallback behavior.

14. Measuring impact

Use the Performance API to measure render time differences. With content-visibility: auto, the initial DOMContentLoaded time can be 40-60% faster on content-heavy pages, especially those with complex layouts below the fold.

15. Summary

content-visibility: auto + contain-intrinsic-size is a powerful, declarative way to optimize rendering of long pages. It requires no JavaScript, degrades gracefully, and can dramatically improve perceived performance.

Pair it with image lazy loading for maximum impact.
