Skeleton Duration Token Bug

Hardcoded 2s shimmer duration ignores the framework token system.

Issue #5760

The Problem

.ease-skeleton uses animation-duration: 2s directly instead of var(--ease-duration-medium). Users cannot customize skeleton shimmer speed via tokens.

Before vs After

Broken — Hardcoded

Fixed 2s Duration

Shimmer speed is always 2s regardless of token overrides.

Fixed — Token-Based

Variable Duration (1s)

Shimmer respects --ease-duration-medium token.
