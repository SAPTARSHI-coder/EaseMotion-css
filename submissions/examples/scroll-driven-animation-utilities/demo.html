The utilities

Three utilities.
One stylesheet.

Drop these classes on any element and the browser handles the rest using native animation-timeline. No IntersectionObserver, no scroll listeners, no polyfills needed.

progress-bar.css 
/* ── Scroll Progress Bar ── */
@keyframes em-progress-grow {
  from { transform: scaleX(0); }
  to   { transform: scaleX(1); }
}

.em-scroll-progress {
  position: fixed;
  top: 0; left: 0; right: 0;
  transform-origin: left center;
  animation: em-progress-grow
    linear both;
  animation-timeline: scroll();
}
fade-reveal.css 
/* ── Scroll Fade Reveal ── */
@keyframes em-fade-in {
  from {
    opacity: 0;
    translate: 0 32px;
  }
  to {
    opacity: 1;
    translate: 0 0;
  }
}

.em-scroll-fade {
  animation: em-fade-in
    ease-out both;
  animation-timeline: view();
  animation-range:
    entry 0% entry 40%;
}
scale-reveal.css 
/* ── Scroll Scale Reveal ── */
@keyframes em-scale-in {
  from {
    opacity: 0;
    scale: 0.82;
  }
  to {
    opacity: 1;
    scale: 1;
  }
}

.em-scroll-scale {
  animation: em-scale-in
    cubic-bezier(0.34,1.56,0.64,1)
    both;
  animation-timeline: view();
  animation-range:
    entry 0% entry 45%;
}

Why it works

Built on the platform.
Not around it.

Scroll-driven animations use the browser's compositor thread — meaning they run off the main thread, stay jank-free, and don't compete with your JavaScript for CPU time.

⚡

Zero JavaScript

No IntersectionObserver, no scroll event listeners, no requestAnimationFrame. The browser does the work natively via animation-timeline.

animation-timeline: scroll()
🎯

Compositor-Driven

Scroll-linked animations are resolved on the compositor thread, keeping your 60 fps locked even when the main thread is busy parsing scripts.

Off main thread
♿

Accessible by Default

Full prefers-reduced-motion support baked in. Elements are always visible — animations are purely progressive enhancement.

prefers-reduced-motion
📐

Precise Range Control

Use animation-range to control exactly when in the viewport an animation starts and finishes — in percentages or absolute lengths.

animation-range: entry 0% entry 40%
📦

Tiny Footprint

The entire utilities stylesheet is under 8 KB unminified. No runtime, no tree-shaking needed — just link and go.

< 8 KB
🔌

Graceful Fallback

Unsupporting browsers receive a clean @supports not block that keeps all content fully visible and functional.

@supports fallback

Usage

Add a class.
That's the whole API.

Copy the stylesheet, add a class, scroll. No configuration objects, no init functions, no wrappers. The platform is the framework.

Progress bar

Fixed element at the top of the page. Automatically fills left to right as the user scrolls the document.

<div class="em-scroll-progress"></div>

Fade reveal

Any block element fades up into view as it enters the viewport. Works on cards, headings, paragraphs, images.

<section class="em-scroll-fade">

Scale reveal

Elements pop into view from 82% → 100% scale with a subtle spring overshoot via cubic-bezier.

<article class="em-scroll-scale">

Combine freely

The utilities are independent — stack them on different elements, mix fade and scale in the same layout.

<div class="em-scroll-fade em-scroll-scale">
