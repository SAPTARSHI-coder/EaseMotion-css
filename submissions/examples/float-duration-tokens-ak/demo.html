Float Duration Token Bug

Hardcoded 3s duration in ease-float ignores the framework token system.

Issue #5757

The Problem

.ease-float sets animation-duration: 3s directly instead of using var(--ease-duration-slow). Users cannot customize float speed via duration tokens.

Fixed 3s Duration

Duration cannot be customized via --ease-duration-* tokens.

Fixed — Token-Based

Duration respects --ease-duration-slow token.
