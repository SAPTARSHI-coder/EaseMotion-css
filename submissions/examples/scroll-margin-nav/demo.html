Scroll Margin & Anchor Navigation

Smooth-scrolling sections with scroll-margin-top offsets to keep content visible beneath a sticky header. Active link highlighting via IntersectionObserver.

01 Introduction

Anchor-based navigation is one of the oldest patterns on the web, but it has always had a friction point: fixed or sticky headers cover the top of the target section. The CSS scroll-margin property solves this natively.

By setting scroll-margin-top: 5rem on each section, the browser automatically adds breathing room when jumping to the anchor — no JavaScript required.

section {
  scroll-margin-top: 5rem;
}

02 Getting Started

Add a sticky header with position: sticky; top: 0. Set scroll-behavior: smooth on html for animated scrolling. Give each section an id and matching scroll-margin-top equal to the header height.

html {
  scroll-behavior: smooth;
}

header {
  position: sticky;
  top: 0;
}

section {
  scroll-margin-top: 5rem;
}

Tip

Use scroll-padding-top on html as a global fallback instead of per-element margins.

03 API Reference

scroll-margin — Shorthand for scroll-margin-top, scroll-margin-right, scroll-margin-bottom, scroll-margin-left. Defines the outer offset used when snapping or scrolling to an element.

scroll-behavior — Accepts auto (instant jump) or smooth (animated). Can be set on html or individual scroll containers.

scroll-padding — Sets the inset of the scrollport. Like padding but for the scroll snap viewport. Works as a container-level alternative to per-element scroll-margin.

/* Global approach — single rule */
html {
  scroll-padding-top: 5rem;
  scroll-behavior: smooth;
}

04 Examples

The scroll-margin values can differ per section. For instance, a tall hero section might need scroll-margin-top: 8rem while a compact FAQ item needs only 3rem.

Combined with scroll-behavior: smooth, the browser handles the entire scroll animation, freeing developers from writing or maintaining custom scroll libraries.

05 FAQ

Does scroll-margin affect layout? No — it only affects scroll snapping and scroll-to-anchor behavior. It does not change the visual box model.

What about browser support? scroll-margin and scroll-behavior are supported in all modern browsers. For older browsers, the page still works — the anchor jumps instantly without the offset.

Should I use scroll-margin or scroll-padding? scroll-padding on the scroll container applies to all children. scroll-margin on individual elements allows per-target customization. Use whichever fits your pattern.
