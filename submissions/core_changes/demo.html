Card Glow Reduced Motion Fix

Bug: .ease-card-glow under prefers-reduced-motion: reduce removes the transition but the hover glow still snaps instantly — a sudden flash that is worse for motion-sensitive users.

Broken transition: none !important

⚡

Glow Card

Hover over me — with reduced motion enabled, the glow snaps on instantly.

The glow appears as a sudden flash (0ms snap) which is jarring for motion-sensitive users.

Fixed box-shadow: none on hover

✨

Glow Card

With reduced motion enabled, the glow is completely removed on hover — no sudden visual change.

No sudden flash. The glow is either always on (if preferred) or entirely removed on hover.

The Fix

Instead of transition: none !important which still allows the hover box-shadow to snap, use one of these approaches: 

/* Option A: Remove the glow entirely on hover when reduced motion is preferred */
@media (prefers-reduced-motion: reduce) {
  .ease-card-glow:hover,
  .ease-card-glow:focus-visible {
    box-shadow: none;
  }
}

/* Option B: Keep the glow always-on so no transition occurs */
@media (prefers-reduced-motion: reduce) {
  .ease-card-glow {
    box-shadow: 0 0 20px rgba(108, 99, 255, 0.3);
  }
}

Option A is recommended — it completely avoids sudden visual changes by removing the glow effect on interaction when the user has requested reduced motion.
